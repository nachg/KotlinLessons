
# Модульное программирование в Kotlin

## Что такое модульное программирование?

Модульное программирование — это подход к разработке, при котором программа разбивается на логические части — **модули**. Каждый модуль решает свою небольшую задачу, а вместе они образуют целое приложение. Это помогает:

- Упростить понимание кода.

- Повторно использовать модули в разных проектах.

- Легче тестировать и отлаживать отдельные части.

- Работать над проектом командой, не мешая друг другу.


В Kotlin основным инструментом для модульности являются **пакеты** (package) и **видимость** (модификаторы доступа). Сегодня мы рассмотрим их на примере функций и свойств верхнего уровня — без классов.

---

## Пакеты (package)

Пакет — это контейнер для кода. Он объединяет связанные функции, переменные и другие объявления. Пакеты позволяют избежать конфликтов имён и структурировать проект.

### Объявление пакета

В начале каждого файла можно указать, к какому пакету он относится:

kotlin

// файл: src/main/kotlin/geometry/rectangle.kt
package geometry
fun perimeter(width: Double, height: Double): Double = 2 * (width + height)
fun area(width: Double, height: Double): Double = width * height

Если пакет не указан, код попадает в **default package** (не рекомендуется для больших проектов).

### Иерархия пакетов

Пакеты могут быть вложенными. Например, `geometry.figures` — это подпакет `geometry`. На диске это соответствует структуре папок.

text

src/
main/
kotlin/
geometry/
rectangle.kt   (package geometry)
circle.kt      (package geometry)
math/
algebra.kt     (package math)

### Использование кода из другого пакета

Чтобы использовать функции из другого пакета, нужно либо указать полное имя, либо сделать импорт.

kotlin

// файл: src/main/kotlin/main.kt
package main
import geometry.area               // импорт конкретной функции
import geometry.perimeter          // можно импортировать несколько
fun main() {
val w = 5.0
val h = 3.0
println("Площадь: ${area(w, h)}")
println("Периметр: ${perimeter(w, h)}")
}

Или без импорта:

kotlin

fun main() {
println("Площадь: ${geometry.area(5.0, 3.0)}")
}

## Открытая и закрытая части модуля

Модуль должен предоставлять внешнему миру только то, что нужно, а внутренние детали скрывать. Это называется **инкапсуляцией**. В Kotlin управлять видимостью помогают модификаторы доступа.

### Видимость для top-level объявлений

Top-level — это объявления (функции, переменные), которые находятся непосредственно в файле, не внутри класса.

|Модификатор|Доступ|
|---|---|
|`public` (по умолчанию)|Видно везде (в любом пакете, в любом модуле).|
|`private`|Видно только в том же файле.|
|`internal`|Видно в пределах одного **модуля компиляции** (например, в одной Gradle-подсистеме).|

**Примечание:** `protected` для top-level объявлений не имеет смысла — он работает только внутри классов.

### Пример

Создадим пакет `geometry` с двумя файлами. Один файл будет содержать открытую часть, другой — внутренние детали.

kotlin

// geometry/rectangle.kt
package geometry
// Открытая функция — public по умолчанию
fun rectangleArea(width: Double, height: Double): Double = width * height
// Вспомогательная функция, скрытая внутри файла
private fun validatePositive(value: Double): Boolean = value > 0
// Используем private функцию внутри файла
fun safeRectangleArea(width: Double, height: Double): Double? {
return if (validatePositive(width) && validatePositive(height)) {
width * height
} else {
null
}
}

kotlin

// geometry/circle.kt
package geometry
import kotlin.math.PI
// public функция (по умолчанию)
fun circleArea(radius: Double): Double = PI * radius * radius
// internal функция — доступна только в пределах модуля компиляции
internal fun circleDiameter(radius: Double): Double = 2 * radius

Теперь в другом пакете:

kotlin

// main.kt
package main
import geometry.rectangleArea
import geometry.safeRectangleArea
// import geometry.validatePositive     // Ошибка: private, недоступен
// import geometry.circleDiameter       // Ошибка: internal, если main.kt в другом модуле
fun main() {
println(rectangleArea(4.0, 5.0))        // OK
println(safeRectangleArea(4.0, 5.0))    // OK
}

Если `main.kt` находится в том же модуле компиляции, `internal` функция `circleDiameter` будет доступна.

## Зачем делить на открытую и закрытую части?

- **Стабильность:** открытая часть — это контракт, который вы обещаете не менять без необходимости. Внутренние детали можно менять свободно.

- **Безопасность:** скрытые данные не могут быть испорчены извне.

- **Упрощение использования:** пользователь видит только нужные функции, не отвлекаясь на вспомогательные.

## Итоги

- **Модуль** в Kotlin — это логическая единица, часто соответствующая пакету.

- **Пакеты** позволяют группировать код и управлять пространством имён.

- **Модификаторы доступа** (`public`, `private`, `internal`) определяют, какие части модуля видны снаружи.

- Скрывая внутренние детали, мы делаем код надёжнее и удобнее для использования.


Следующий шаг — познакомиться с классами, где концепция инкапсуляции раскроется ещё глубже.
