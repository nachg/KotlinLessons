# Классы в Kotlin: тип и модуль

В предыдущей лекции мы говорили о модульном программировании с помощью пакетов и функций верхнего уровня. Теперь познакомимся с главной конструкцией, которая объединяет данные и поведение, — **классом**.  
Класс в Kotlin, как и в других объектно-ориентированных языках, выполняет две ключевые роли, сформулированные Бертраном Майером (автором языка Eiffel):

1. **Класс как тип** — он определяет набор значений (состояние) и операций (поведение), которыми обладают объекты этого типа.

2. **Класс как модуль** — он служит единицей структурирования программы, объединяя связанные элементы и управляя доступом к ним (инкапсуляция).


Рассмотрим эти роли на примерах Kotlin.

---

## 1. Класс как тип

Когда мы говорим «класс — это тип», мы имеем в виду, что класс задаёт **абстракцию данных**: описывает, из каких частей (свойств) состоит объект и какие действия (методы) с ним можно выполнить. После определения класса мы можем создавать переменные этого типа, передавать их в функции и т.д.


```kotlin
// Класс Rectangle определяет тип "прямоугольник"
class Rectangle(val width: Double, val height: Double) {
    // Метод, доступный для всех объектов типа Rectangle
    fun area(): Double = width * height
    fun perimeter(): Double = 2 * (width + height)
}
fun main() {
    val rect: Rectangle = Rectangle(5.0, 3.0)  // rect имеет тип Rectangle
    println(rect.area())      // 15.0
    println(rect.perimeter()) // 16.0
}
```
Здесь `Rectangle` — это **тип**. Переменная `rect` может хранить только объекты, соответствующие этому описанию. Любой объект `Rectangle` обладает полями `width`, `height` и методами `area`, `perimeter`.

### Почему это важно?

- **Безопасность типов** — компилятор проверяет, что мы не пытаемся вызвать `area()` у строки или числа.

- **Единое описание** — один раз определили, что такое прямоугольник, и используем это описание везде.

- **Абстракция** — пользователь класса работает только с методами, не вникая в детали вычислений.


---

## 2. Класс как модуль

Майер подчёркивал, что класс — это не только тип, но и **модуль**, то есть логическая единица программы, которая имеет **открытую часть** (интерфейс) и **закрытую часть** (реализацию). В Kotlin, как и в предыдущей лекции, мы управляем видимостью с помощью модификаторов доступа, но теперь они применяются к членам класса.

### Открытая часть (интерфейс)

Это то, что класс обещает внешнему миру: публичные свойства и методы. Клиенты могут их вызывать, не зная, как они устроены внутри.

### Закрытая часть (реализация)

Внутренние детали, которые скрыты от клиентов: приватные поля, вспомогательные методы. Они могут меняться без влияния на код, использующий класс.


```kotlin

// Класс как модуль: скрываем детали вычисления скидки
class DiscountCalculator(private val rate: Double) {
    // Открытая часть (интерфейс)
    fun apply(price: Double): Double {
        val discounted = price * (1 - rate)
        return roundToTwoDecimals(discounted)
    }
    // Закрытая часть (реализация)
    private fun roundToTwoDecimals(value: Double): Double {
        return (value * 100).toInt() / 100.0
    }
}
fun main() {
    val calc = DiscountCalculator(0.25) // 25% скидка
    println(calc.apply(100.0))          // 75.0
    // calc.roundToTwoDecimals(10.0)    // Ошибка: private, недоступен
}
```


В этом примере:

- `rate` — приватное свойство, установлено при создании объекта.

- Метод `apply` — публичный, клиенты используют его для расчёта цены со скидкой.

- `roundToTwoDecimals` — приватный метод, деталь реализации, скрыта от клиента.


Благодаря этому:

- **Можно изменить реализацию** (например, округлять по-другому), не трогая код, который вызывает `apply`.

- **Клиент не зависит от внутренностей**, что снижает связанность кода.

## Сочетание ролей: тип + модуль

В реальном проекте класс одновременно выступает и как тип, и как модуль. Как тип он предоставляет единый интерфейс для работы с данными, а как модуль — инкапсулирует сложность.

Рассмотрим более полный пример:

```kotlin

// Банковский счёт: тип (Account) и модуль
class Account(private val owner: String, initialBalance: Double) {
    // Открытые свойства (только чтение)
    val balance: Double
        get() = _balance  // публичный геттер, поле скрыто
    private var _balance = initialBalance  // скрытое поле
    // Открытые методы
    fun deposit(amount: Double) {
        require(amount > 0) { "Сумма депозита должна быть положительной" }
        _balance += amount
    }
    fun withdraw(amount: Double): Boolean {
        if (amount <= 0 || amount > _balance) return false
        _balance -= amount
        return true
    }
    fun displayInfo() {
        println("Владелец: $owner, Баланс: $balance")
    }
    // Вспомогательный приватный метод
    private fun log(operation: String) {
        println("Лог: $operation для $owner")
    }
}
fun main() {
    val acc: Account = Account("Иван", 1000.0)  // тип Account
    acc.deposit(500.0)
    acc.withdraw(200.0)
    acc.displayInfo()
    // println(acc._balance)  // Ошибка: private
}
```

Здесь:

- **Тип** `Account` задаёт набор операций (`deposit`, `withdraw`, `displayInfo`) и состояние (`balance`).

- **Модуль** скрывает детали (поле `_balance`, метод `log`) и предоставляет стабильный интерфейс.


---

## Почему класс — это естественное развитие модульности?

В предыдущей лекции мы использовали пакеты и функции верхнего уровня. Такой подход тоже позволяет разбить программу на модули, но у него есть недостатки:

- Функции оперируют данными, которые передаются как параметры, что часто приводит к дублированию кода.

- Сложно обеспечить инварианты (например, баланс не может быть отрицательным), потому что данные не привязаны к функциям.


Класс же связывает данные и методы в одну сущность, делая модуль **целостным** и **самоуправляемым**.

---

## Итоги

- **Класс как тип** — описывает структуру данных и операции над ними, позволяет компилятору контролировать типы и обеспечивает абстракцию.

- **Класс как модуль** — инкапсулирует реализацию, скрывая внутренние детали за открытым интерфейсом, что упрощает поддержку и изменения.

- В Kotlin эти две роли реализуются через синтаксис класса, модификаторы доступа (`private`, `public` по умолчанию, `internal`, `protected`) и свойства.


Понимание классов в таком дуализме поможет вам проектировать надёжные, гибкие и понятные программы — именно так, как задумывалось в объектно-ориентированном подходе.
